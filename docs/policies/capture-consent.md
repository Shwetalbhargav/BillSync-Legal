# Gmail And Desktop Capture Consent

Before enabling Gmail, research, web timer, or desktop agent capture, the workspace must disclose:
- What activity is captured.
- What is not captured.
- Who can view captured entries.
- How users can pause, discard, edit, or mark work non-billable.
- How offline queue and duplicate-source protection behave.

Desktop capture records activity metadata required for work classification. It must not capture keystroke values, screenshots, page text, or document contents unless a future explicit feature and consent flow adds that capability.

Consent should be recorded per user and per workspace before capture is enabled.
