# Lexora API Test Automation

Production-ready Postman/Newman API automation generated from `lexora_api_inventory.xlsx`.

## Folder Structure

```text
api-test-project/
├── package.json
├── README.md
├── scripts/
│   ├── generate-postman-collection.js
│   ├── parse-api-inventory.js
│   └── validate-inventory.js
├── postman/
│   ├── lexora-api-tests.postman_collection.json
│   ├── staging.postman_environment.json
│   └── production-smoke.postman_environment.json
├── data/
│   └── test-data.json
├── reports/
│   └── .gitkeep
└── .github/workflows/
    └── api-tests.yml
```

## Assumptions

- The workbook is read from `../outputs/api_inventory/lexora_api_inventory.xlsx` by default.
- Override the workbook path with `API_INVENTORY_FILE=/path/to/lexora_api_inventory.xlsx`.
- `GET` requests are treated as read-only unless the path looks action-oriented.
- `POST`, `PUT`, `PATCH`, and `DELETE` requests are marked destructive and skipped unless `runDestructiveTests=true`.
- Protected endpoints receive Bearer auth with `{{token}}`; known public endpoints use no auth.
- JSON Schema tests are lightweight and generated only when `Output Expected` exposes a clear object-like shape.

## Setup

```bash
npm install
npm run generate
```

## Environment Variables

Set these in Postman, Newman, or CI:

- `baseUrl`: API base URL, for example `https://billsync-legal.onrender.com`
- `token`: Bearer token. Can be supplied directly or generated by the login request.
- `loginUsername`: Test account username/email.
- `loginPassword`: Test account password.
- `responseTimeLimitMs`: Shared response-time threshold. Default is `3000`.
- `runDestructiveTests`: Must be `true` to execute mutating requests.

The generator also adds variables for path and query parameters found in the workbook, such as `userId`, `clientId`, `caseId`, and `invoiceId`.

## Commands

```bash
npm run validate:inventory
npm run generate
npm run test:staging
npm run test:production-smoke
```

Production smoke runs only the `Smoke` folder:

```bash
newman run postman/lexora-api-tests.postman_collection.json \
  -e postman/production-smoke.postman_environment.json \
  --folder Smoke
```

## Safety Guidance

- Keep `runDestructiveTests=false` for production.
- Use isolated test accounts and disposable test data for staging full-suite runs.
- Do not commit real JWTs, passwords, API keys, or production customer IDs.
- Populate `data/test-data.json` and environment files with placeholder-safe values only.
- Prefer creating cleanup endpoints or teardown scripts before enabling destructive staging runs in CI.

## CI

The GitHub Actions workflow:

1. Runs on pull requests.
2. Installs dependencies.
3. Regenerates the Postman collection from Excel.
4. Runs staging Newman tests only when staging secrets exist.
5. Uploads Newman reports as artifacts.

Required staging secrets:

- `STAGING_BASE_URL`
- `STAGING_LOGIN_USERNAME`
- `STAGING_LOGIN_PASSWORD`

Secrets are passed to Newman with `--env-var` and are not printed by the workflow.
